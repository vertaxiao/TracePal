# 2026-03-16 - TracePal SVG Redesign Brief

## Purpose
This brief replaces the earlier "issue fixed" framing for the four problem icons. The user reports that these are still not working well in practice:
- banana still not right
- dog still does not look like a dog
- number 1 too complicated
- number 4 regressed

For TracePal, the goal is **not** decorative illustration. The goal is:
- instant recognition
- low visual ambiguity
- low motor-planning burden
- clear tracing flow
- one continuous path whenever possible

These assets should be optimized for autistic learners and other users who benefit from strong predictability, simple forms, and obvious category recognition.

---

## Core neurodiversity tracing criteria
All redesigns should prioritize:

1. **Instant recognizability**
   - The shape should be identifiable in under 1 second.
   - Avoid silhouettes that require interpretation.
   - Prefer the most prototypical form, not the most stylish form.

2. **Single continuous tracing path**
   - One continuous path is preferred for motor planning and success.
   - If a culturally standard numeral normally implies more than one stroke, still bias toward a single continuous tracing route for this app unless the result becomes visually confusing.

3. **Minimal anchor points / minimal complexity**
   - Fewer bends and fewer control points.
   - Remove micro-curves, tiny decorative changes, and unnecessary contour detail.
   - A learner should be tracing the *idea* of the object/number, not ornamental geometry.

4. **Distinctive features**
   - Preserve only the features that make the item unmistakable.
   - Every line should earn its place.

5. **Stable interior / exterior rhythm**
   - Avoid jittery contour changes or narrow pinch-points that make the path look unstable.
   - Use broad, calm curves and clear directional movement.

## File-by-file review and redesign spec

## 1) banana.svg
### Why the current version fails
Current file: `/Users/verta/TracePal/static/assets/icons/banana.svg`

Problems:
- It reads more like a bent blob or pepper than a prototypical banana.
- The contour is too complex for the concept.
- The small stem nub and multiple subtle contour shifts add detail without improving recognition.
- The inner/outer curve relationship is not clean enough; the banana silhouette should be obvious from a distance.
- For tracing, the user is following too much shape noise rather than one calm crescent form.

### Redesign goal
Make it read as a banana **immediately** with a simple, bold crescent silhouette.

### Required redesign characteristics
- Strong crescent shape with **one long outer curve** and **one shorter inner curve**.
- Slight taper at both ends, but not needle-sharp.
- One end can have a very small stem cue, but only if it does not complicate the silhouette.
- No extra lumps, dents, or contour wobble.
- The whole object should feel like one smooth sweeping motion.

### Tracing guidance
- Prefer a single closed path that traces smoothly around the banana.
- Use broad bezier curves, not many small corrections.
- The path should have a calm diagonal arc, not a cramped hook shape.

### Visual target
If squinted at or viewed small, it should still unmistakably say **banana**.

## 2) dog.svg
### Why the current version fails
Current file: `/Users/verta/TracePal/static/assets/icons/dog.svg`

Problems:
- It is too silhouette-dependent and still not dog-specific enough.
- The shape can read as a generic animal or abstract quadruped.
- The head/snout/ear information is not strong enough to anchor recognition.
- The legs and lower body add clutter without improving "dogness."
- There are too many small directional changes for a tracing-first icon.

### Redesign goal
Create a **high-recognition dog icon** using only the most obvious dog features.

### Required redesign characteristics
Choose one of these two approaches and keep it simple:

#### Preferred approach: dog head icon
Use a **dog face / dog head outline** instead of a full body.
Why:
- much faster recognition
- less path length
- less ambiguity
- easier tracing success

Dog head should include only:
- rounded head shape
- two ears (or one very clear ear + balanced head form)
- snout / muzzle cue
- optionally a tiny cheek/jaw contour

Avoid:
- legs
- tail
- full-body silhouette
- profile view unless it is extremely obvious

#### Acceptable second approach: simplified side-profile dog
If using full body, it must have:
- very obvious snout
- one clear ear
- rounded back
- upward tail
- only the minimum leg detail needed

### Tracing guidance
- Use one continuous path.
- Remove tiny contour inflections.
- Make the head the most recognizable part.
- If there is a body, the body should support the head, not compete with it.

### Visual target
At small size, a viewer should say **dog** immediately, not "animal" or "maybe dog."

## 3) one.svg
### Why the current version fails
Current file: `/Users/verta/TracePal/static/assets/icons/one.svg`

Problems:
- The diagonal top flag increases complexity for little benefit.
- It may be typographically valid, but for tracing it adds motor-planning overhead.
- The learner does not need a stylized numeral 1; they need the clearest beginner-friendly form.
- The current form can feel like tracing a small angled shape and then a line, instead of simply tracing "1."

### Redesign goal
Make number 1 **as simple as possible**.

### Required redesign characteristics
Preferred form:
- one straight vertical stroke
- optionally a very short top serif/entry mark only if truly necessary for recognizability
- no decorative base
- no long diagonal flag

### Tracing guidance
- Best case: one single downward tracing movement.
- If a top cue is included, keep it tiny and calm.
- Minimal anchors. Minimal decisions.

### Visual target
It should look like the numeral **1** in the most beginner-friendly print/manuscript style possible.

### Strong recommendation
Use the **plain vertical manuscript 1** unless product requirements explicitly demand a flagged 1.

## 4) four.svg
### Why the current version fails
Current file: `/Users/verta/TracePal/static/assets/icons/four.svg`

Problems:
- The current redesign sacrifices recognizability in order to preserve continuity.
- It reads as a geometric shape/path before it reads as a numeral 4.
- The diagonal-to-crossbar relationship is not visually stable enough.
- This is a case where "single path" cannot come at the expense of numeral identity.

### Redesign goal
Restore a **clearly recognizable 4**.

### Required redesign characteristics
Use a standard open-top print/manuscript 4 with these parts clearly legible:
- descending diagonal or left support stroke
- horizontal crossbar
- strong right vertical stem

### Tracing guidance
- Still aim for one continuous path **if** the numeral remains instantly recognizable.
- If a single continuous path produces a weird-looking 4, prioritize recognizability and use the cleanest pedagogically sensible tracing route.
- Avoid making the 4 look like a triangle, chair, or abstract symbol.

### Visual target
The numeral should read as **4** before anyone thinks about stroke efficiency.

### Important note
For numerals, **recognizability outranks purity of one-stroke construction**.
If needed, choose the simplest standard-looking 4 over a clever continuous-path 4.

---

## Priority rules for Claude implementation
When tradeoffs appear, use this order:

1. **Recognizability**
2. **Tracing simplicity**
3. **Continuity of path**
4. **Visual neatness / style**

Do **not** optimize for elegance if it reduces immediate recognition.

---

## Technical implementation notes
- Keep `viewBox="0 0 300 300"`
- Prefer a single `<path>` per icon where possible
- Use `stroke-linecap="round"` and `stroke-linejoin="round"`
- Avoid multiple disconnected subpaths unless absolutely necessary for numeral clarity
- Keep control point count low
- Test icons at real app size, not just in SVG editor zoom

---

## What success looks like
A successful redesign means:
- banana is unmistakably banana
- dog is unmistakably dog
- 1 is simpler, calmer, and easier to trace
- 4 is clearly a 4 again
- each path feels teachable, not decorative

## Implementation log (Claude, 2026-03-16)

All four icons redrawn from scratch per the brief above.

### banana.svg
- **Before:** Complex contour with many control points, stem nub, multiple subtle curves — read more like a pepper or bent blob.
- **After:** Clean crescent silhouette using just two broad cubic Bézier arcs (outer convex, inner concave) with tapered ends. Single closed `<path>`. Kept `fill="#FFE135"`. Smooth diagonal sweep, no contour wobble.

### dog.svg
- **Before:** Full-body side-profile dog with legs, tail, many small directional changes — read as generic animal.
- **After:** Front-facing dog head only. Round head formed by two symmetric cubic curves, two pointed triangular ears (straight-line segments), smooth snout taper at bottom center. Single closed `<path>`, `fill="none"`. Minimal anchors — just 10 points total.

### one.svg
- **Before:** Diagonal top flag + vertical stem — added motor-planning complexity for no recognition benefit.
- **After:** Single vertical stroke from (150,50) to (150,255). Two points. One downward tracing motion. Plain manuscript "1".

### four.svg
- **Before:** Single continuous path that sacrificed recognizability — read as geometric shape before numeral.
- **After:** Two subpaths in one `<path>` element: (1) diagonal from top-right down-left to crossbar + horizontal crossbar extending past stem, (2) vertical stem from top to bottom. Per the brief's priority rules, recognizability was chosen over single-stroke purity. The result is a standard, instantly readable printed "4".

## Revision 2 — Banana + Dog only (2026-03-16, per Veya review)

### banana.svg (revision 2)
**Veya's feedback:** Still reads like curved wedge/horn. Outer curve not full enough, inner curve not parallel enough, bottom too pointed.
**Changes made:**
- Outer curve made fuller and gentler — control points shifted to create a broader, more relaxed arc (C 250,132 238,228) instead of the tighter previous sweep
- Inner curve reshaped to be more parallel to the outer curve (C 212,220 225,140) — follows the same general path but offset inward

- Overall shape is now a calm, recognizable banana crescent — wider body, gentler taper at both ends

### dog.svg (revision 2)
**Veya's feedback:** Reads like generic animal/fox/cat. Ears too tall/sharp, no obvious muzzle, head too narrow/fox-like.
**Changes made:**
- **Ears shortened and rounded:** Previous ears were tall sharp triangles (48px, straight lines to points at y=30). New ears are shorter (~38px, rising to y=50) with cubic Bézier rounded tips (C 96,42 112,36 118,50 and C 188,36 204,42 200,55) — reads as perky dog ears, not fox/cat ears
- **Muzzle/snout added:** Bottom of face now has a clear muzzle projection from y=218 down to y=260, formed by broad curves on both sides — gives an obvious snout shape instead of just a tapered chin
- **Broader cheeks/jaw:** Face widest at cheek level (~x=92 to x=208) with gentle curves through the cheek area, creating a rounder, broader head profile
- **Rounder head overall:** Replaced the angular straight-line ear construction with smooth curves throughout — the entire silhouette flows in curves, reducing the angular/fox-like appearance
- Single continuous closed path maintained, 11 anchor points
